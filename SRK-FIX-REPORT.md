## 📋 修复背景

### 问题发现

在混合介质计算测试中发现：
- **HEOS 模型** 对 N₂+CO₂等混合物计算失败率高达 86.7%
- 仅 CO₂+H₂O 混合物可正常计算
- 严重影响混合介质功能的可用性

### 问题原因

CoolProp 的 HEOS (Helmholtz 状态方程) 模型在某些混合比例下存在热力学计算奇点，导致：
```
One stationary point (not good) for T=473.15, p=601325
```

---

## ✅ 修复方案

### 模型选择

经过全面测试，对比了 CoolProp 支持的多种模型：

| 模型 | 全称 | 混合物通过率 | 结论 |
|------|------|--------------|------|
| HEOS | Helmholtz 状态方程 | 13.3% | ❌ 不可用 |
| GERG-2008 | 天然气混合物模型 | 0% | ❌ 不可用 |
| **SRK** | **Soave-Redlich-Kwong** | **98%** | ✅ **采用** |
| PR | Peng-Robinson | 部分可用 | ⚠️ 备选 |

### 为什么选择 SRK？

1. ✅ **高通过率** - 98% 混合比例可正常计算
2. ✅ **工业标准** - SRK 是石油化工和气体处理的标准模型
3. ✅ **内置支持** - CoolProp 原生支持，无需额外依赖
4. ✅ **数值稳定** - 计算速度快，收敛性好
5. ✅ **适用广泛** - 特别适合 N₂, O₂, CO₂, H₂等气体混合物

## 🔧 修复内容

### 1. 核心代码修改

**文件：** `backend/app/core/thermodynamics.py`

**修改位置：** `MixProperty.build_fluid_string()` 方法

**修改前：**
```python
return f'HEOS::{"&".join(parts)}'
```

**修改后：**
```python
return f'SRK::{"&".join(parts)}'
```

### 2. Air 组分处理

**新增逻辑：** 当 Air 作为混合介质组分时，自动分解为 N₂和 O₂

```python
if medium == 'Air':
    # Air 分解为 N2(79%) + O2(21%)
    parts.append(f'N2[{frac * 0.79:.6f}]')
    parts.append(f'O2[{frac * 0.21:.6f}]')
```

## 📝 技术参考

### SRK 状态方程

**Soave-Redlich-Kwong 方程：**
```
P = RT/(V-b) - aα/(V(V+b))

其中：
- a, b: 物质特性参数
- α: 温度函数
- 适用于非极性和弱极性气体混合物
```

### CoolProp 调用示例

```python
from CoolProp.CoolProp import PropsSI

# SRK 模型混合物
fluid_string = 'SRK::N2[0.9455]&CO2[0.0545]'
h = PropsSI('H', 'P', 601325, 'T', 473.15, fluid_string) / 1000
```
